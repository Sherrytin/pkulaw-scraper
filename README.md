## 📌 项目背景

本项目旨在高效、稳定地采集北大法宝公开政策数据，满足大规模学术研究的需要。  
**少量政策**可直接通过配套的 **GUI 工具** 快速获取；**大量政策**（如全年几十万条）则推荐使用本仓库的命令行脚本，利用**多分片并行 + 断点续传**自动完成，几乎无需人工干预。

如果你的学校已购买北大法宝库，**校园网 IP 可直接访问**，无需额外登录。

> ⚠️ **重要声明**

## ✨ 功能特点

- **🛡️ 极低风控**：单分片顺序请求 + 随机延时，模拟真实用户行为，有效规避验证码。
- **⏯️ 断点续传**：自动记录已成功处理的 URL，中断后可无缝继续，绝不重复。
- **⚡ 多分片并行**：将一个月的数据智能切分为 2~5 个独立分片，多进程/多实例并发，线性提升速度（总速率仍控制在安全阈值内）。
- **📅 按月自动调度**：提供调度脚本，可顺序处理 3 月至 12 月，每个月份独立运行且支持超时自动重启（90 分钟）。
- **🖥️ 双模式选择**：
  - **命令行版**（`split_monthly_v2.py`）：适合服务器或后台批量运行，支持无头模式，资源占用低。
  - **GUI 工具**（`PkulawCrawlerGUI.exe`）：图形界面操作，实时查看日志，适合小规模测试或手动干预。
- **🧹 数据完整性保证**：原子写入 JSON 文件 + 自动备份，即使进程被强制终止，已保存的数据也不会损坏。

## 🚀 快速开始

### 环境要求
- Windows 10/11（GUI 工具为 Windows 专用）
- Python 3.8+（命令行版）
- Chrome 浏览器（由 DrissionPage 自动调用）
- 校园网等机构网络或个人会员（可访问北大法宝）

### 安装与配置

#### 方式一：使用命令行爬虫（推荐大规模采集）

1. **克隆仓库**
   ```bash
   git clone https://github.com/Sherrytin/pkulaw-scraper.git
   cd pkulaw-scraper
2. **安装依赖**
   ```bash
   pip install -r requirements.txt
3. **准备 URL 文件**
   将各月份的 URL 列表放入 D:\python\data\policy\2025\ 目录（可在 split_monthly_v2.py 中修改路径。
4. **运行单个月份测试**
   ```bash
   python split_monthly.py --year 2025 --month 10t
5. **自动处理所有月份（推荐）**
   ```bash
   python run_scheduler.py
调度脚本会自动从指定起始月份开始，每 90 分钟重启一次当前月份（利用断点续传），直到该月份所有 URL 处理完毕，然后自动进入下一个月。

#### 方式二：使用 GUI 工具（适合小批量）
如果你不熟悉命令行，或者只需要爬取少量政策，可以使用我们提供的图形界面工具。

1. **下载 GUI 工具**

- 从 Releases 页面下载 PkulawCrawlerGUI.exe。

- 无需安装 Python 环境，双击即可运行。

2. **学习使用**

- 详细操作教程：[B站视频](https://www.bilibili.com/video/BV12MnizLENM/)

3. **GUI 工具主要功能**

- 收集 URL：手动在浏览器中导航到目标列表页（如搜索“知识产权”），工具会自动提取当前页面的所有政策链接，保存到 urls.txt。

- 下载内容：读取 urls.txt 中的 URL，依次访问详情页并提取全文和元数据，内容输出txt文件，保存到指定文件夹。

- 实时日志：显示爬取进度、成功/失败信息。

- 延时设置：可自定义请求间隔，避免触发风控。

4. **注意事项**

- GUI 工具为单线程版本，适合测试或小批量采集（建议 < 5000 条）。

- 对于全年数十万条的大规模任务，请改用方式一的命令行分片脚本，效率更高且支持断点续传。

- 使用前请确保网络可访问北大法宝，且校园网已认证。
  
## 📂 输出结果字段说明

无论使用哪种方式，最终生成的 JSON 文件均包含以下主要字段：

| 字段 | 说明 |
|------|------|
| `title` | 政策标题 |
| `detail_url` | 政策唯一标识（url） |
| `province` | 省份 |
| `EffectivenessDic` | 效力位阶 |
| `TimelinessDic` | 时效性（“现行有效”映射为“有效”） |
| `IssueDate` | 年份（如 `2025`） |
| `category_1` | 法规类别 |
| `pub_depart` | 制定机关 |
| `pub_num` | 发文字号 |
| `pub_date` | 公布日期（加“公布”后缀） |
| `use_date` | 施行日期（加“施行”后缀） |
| `is_time` | 时效性原文 |
| `detail_flag` | 正文全文 |
| `add_time` | 数据添加时间 |

## ⚙️ 配置参数说明

在 `split_monthly.py` 中可调整以下参数：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `NUM_SPLITS` | 分片数（并行实例数） | 3（2~4） |
| `MIN_DELAY` / `MAX_DELAY` | 每次请求随机延时（秒） | 2.0 / 4.0 |
| `HEADLESS` | 是否无头模式 | `True` |
| `BACKUP_INTERVAL` | 每成功 N 条备份一次 | 100 |
| `BASE_DIR` | URL 文件所在目录 | 根据实际修改 |

在 `run_scheduler.py` 中可调整：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `YEAR` | 年份 | 2025 |
| `START_MONTH` / `END_MONTH` | 起始/结束月份 | 3 / 12 |
| `TIMEOUT_SECONDS` | 单月份超时时间（秒） | `90 * 60` |

## 🧪 断点续传示例

- 运行过程中按 `Ctrl+C` 或调度脚本超时自动终止，已成功的数据会立即保存。
- 重新运行同一命令，脚本会自动跳过已完成的 URL，继续处理剩余部分。
